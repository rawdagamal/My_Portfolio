Project 7: VTP and VLAN Configuration with EtherChannel and Port Security

Objective:

The objective of this project is to set up a network that demonstrates the functionality of VLAN Trunking Protocol (VTP), VLANs, EtherChannel, and Port Security. The network includes multiple switches configured with different roles and VLAN assignments.

Requirements:

  • Devices:
    • Switches
    • PCs
    • Ethernet cables

    Network Topology

    Server Switch (SW1):

    • Configured as VTP Server
    • VLANs: 10, 20, 30, 40, 50, 60
    • EtherChannel on interfaces G1 and G2
    • SSH enabled for management

    Client Switches (SW2 and SW3):

    • Configured as VTP Clients
    • EtherChannel on interfaces to Server Switch
    • VLANs pushed from Server Switch
    • Specific VLANs assigned to ports
    • Port Security enabled on access ports

    PCs:

    • Connected to respective VLANs

    Steps to Configure

    1. Configure VTP on Server Switch (SW1):

    
          Switch1(config)# vtp mode server
          Switch1(config)# vtp domain mydomain
          Switch1(config)# vtp password mypassword

    2. Create VLANs on Server Switch (SW1):

    
          Switch1(config)# vlan 10
          Switch1(config-vlan)# name Management
          Switch1(config)# vlan 20
          Switch1(config-vlan)# name Sales
          Switch1(config)# vlan 30
          Switch1(config-vlan)# name Engineering
          Switch1(config)# vlan 40
          Switch1(config-vlan)# name HR
          Switch1(config)# vlan 50
          Switch1(config-vlan)# name Finance
          Switch1(config)# vlan 60
          Switch1(config-vlan)# name Admin

    3. Configure EtherChannel on Server Switch (SW1):

    
          Switch1(config)# interface range f0/1-4
          Switch1(config-if-range)# channel-group 1 mode active
          Switch1(config-if-range)#switchport mode trunk
          Switch1(config-if)# exit

    3. Configure EtherChannel on Server Switch (SW1):

    
          Switch1(config)# interface range f0/5-8
          Switch1(config-if-range)# channel-group 2 mode active
          Switch1(config-if-range)#switchport mode trunk
          Switch1(config-if)# exit

    4. Configure VTP on Client Switches (SW2 and SW3):

    
          Switch(config)# vtp mode client
          Switch(config)# vtp domain mydomain
          Switch(config)# vtp password mypassword

    5. Configure EtherChannel on Client Switches (SW2 and SW3):

    
          Switch1(config)# interface range f0/1-4
          Switch1(config-if-range)# channel-group 1 mode passive
          Switch1(config-if-range)# switchport mode trunk
          Switch2(config)# interface range f0/1-4
          Switch2(config-if-range)# channel-group 2 mode passive
          Switch2(config-if-range)# switchport mode trunk

    6. Assign VLANs to Ports on Client Switches:

    
          Switch(config)# interface range fa0/1 - 2
          Switch(config-if-range)# switchport mode access
          Switch(config-if-range)# switchport access vlan 10
          Switch(config)# interface range fa0/3 - 4
          Switch(config-if-range)# switchport mode access
          Switch(config-if-range)# switchport access vlan 20

    7. Configure Port Security on Access Ports:

    
          Switch(config)# interface fa0/1
          Switch(config-if)# switchport port-security
          Switch(config-if)# switchport port-security maximum 1
          Switch(config-if)# switchport port-security violation restrict
          Switch(config-if)# switchport port-security mac-address sticky

    8. Enable SSH on Server Switch:

    
          Switch(config)# ip domain-name mydomain.com
          Switch(config)# crypto key generate rsa
          Switch(config)# username admin password mypassword
          Switch(config)# line vty 0 4
          Switch(config-line)# login local
          Switch(config-line)# transport input ssh

    Verification

    1. Verify VTP Status:

    
          Switch# show vtp status

    2. Verify VLANs on Client Switches:

    
          Switch# show vlan brief

    3. Verify EtherChannel Status:

    
          Switch# show etherchannel summary

    4. Verify Port Security:

    
          Switch# show port-security interface fa0/1

    5. Verify SSH Access:

    Attempt to SSH into the Server Switch using an SSH client:

    
          ssh admin@<server_switch_ip>

    Verification Questions

    1. Did you enable VTP on the main switch (SW1)?
        YES
      Check with the command: show vtp status

    2. Did you create VLANs on the main switch (SW1)?
        YES
      Check with the command: show vlan brief

    3. Did you configure EtherChannel on the main switch (SW1)?
        YES
      Check with the command: show etherchannel summary

    4. Did you configure VTP on the client switches (SW2 and SW3)?
        YES
      Check with the command: show vtp status

    5. Did you configure EtherChannel on the client switches (SW2 and SW3)?
        YES
      Check with the command: show etherchannel summary

    Project 5: Setting Up a Small Office Network with EtherChannel

    Objective:

    Design and implement a small office network using EtherChannel to increase bandwidth and redundancy between switches. Configure static IP addresses and DHCP for network devices.

    Requirements:

    • Devices:
      • Router
      • three Switches
      • Three Personal Computers (PCs)
    • Small Office Network Setup
    • Configurations:
      • Use EtherChannel to aggregate links between switches.
      • Assign static IP addresses to each PC.
      • Configure DHCP on the router for dynamic IP address allocation.

    Steps

  • Step 1: Physical Network Setup
    • Connect PCs to Switch 1:
      • Connect PC1, PC2, and PC3 to Switch 2 using Ethernet cables.
    • Connect Switches with EtherChannel:
      • Connect Switch 1 to Switch 2 using two or more Ethernet cables to create an EtherChannel.
      • and Connect Switch 1 to Switch 3 using two or more Ethernet cables to create an EtherChannel.
    • Connect Switch 1 to the Router:
      • Use an Ethernet cable to connect Switch 1 to the router.
  • Step 2: Configure EtherChannel on Switches
    • Create EtherChannel on Switch 1:
      • 
            Switch1> enable
      
            Switch1# configure terminal
      
            Switch1(config)# interface range F0/1 - 4
      
            Switch1(config-if)#switchport mode trunk 
      
            Switch1(config-if-range)# channel-group 1 mode active
      
            Switch1(config-if-range)# exit
      
            Switch1(config)# interface range F0/5-8
      
            Switch1(config-if)#switchport mode trunk 
      
            Switch1(config-if-range)# channel-group 2 mode active
      
            Switch1(config-if-range)# exit
    • Create EtherChannel on Switch 2:
      • 
            Switch2> enable
      
            Switch2# configure terminal
      
            Switch2(config)# interface range F0/1 - 4
      
            Switch2(config-if)#switchport mode trunk
       
            Switch2(config-if-range)#channel-group 1 mode passive 
      
            Switch2(config-if-range)# exit
    • Create EtherChannel on Switch 2:
      • 
            Switch3> enable
      
            Switch3# configure terminal
      
            Switch3(config)# interface range F0/1 - 4
      
            Switch3(config-if)#switchport mode trunk
       
            Switch3(config-if-range)#channel-group 2 mode passive 
      
            Switch3(config-if-range)# exit
            Switch3(config)# interface  F0/5
      
            Switch3(config-if)#switchport mode trunk

    Note: The configuration above uses LACP (Link Aggregation Control Protocol) with mode active. You can also use mode on for a static EtherChannel.

  • Step 3: Router Configuration
    • Assign IP Address to the Router:
      • 
            Router> enable
      
            Router# configure terminal
      
            Router(config)# interface GigabitEthernet0/0/0
      
            Router(config-if)# ip address 192.168.1.1 255.255.255.0
      
            Router(config-if)# no shutdown
      
            Router(config-if)# exit
  • Step 4: Configure DHCP on the Router
    • Enter DHCP Configuration Mode:
      • 
            Router(config)# ip dhcp pool OFFICE
      
            Router(dhcp-config)# network 192.168.1.0 255.255.255.0
      
            Router(dhcp-config)# default-router 192.168.1.1
      
            Router(dhcp-config)# dns-server 8.8.8.8
      
            Router(dhcp-config)# exit
  • Step 5: Verify the Solution
    • Ensure Physical Connections:
      • Verify that all PCs are connected to Switch 1 and that the switches are connected with EtherChannel to the router.
    • Check IP Configuration:
      • Confirm that each PC has the correct Dynamic IP address assigned.
    • Test Connectivity:
      • 
            ping 192.168.1.1
    • Verify EtherChannel:
      • On each switche, verify the EtherChannel status.
      • 
            Switch# show etherchannel summary

    Verification Questions:

    • Are all PCs correctly connected to the network?
      • YES
    • Have all PCs received the correct IP addresses (static or dynamic)?
      • YES
    • Can the PCs connect to the router using the ping command?
      • YES
    • Is the EtherChannel successfully established between the switches?
      • YES

Router1(config)# exit

Router2> enable

      1. Enter the configuration mode on the router: 
        
Router> enable
Router# configure terminal
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit
      2. Set up DHCP: 
        
Router(config)# ip dhcp pool OFFICE
Router(dhcp-config)# network 192.168.1.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.1.1
Router(dhcp-config)# dns-server 8.8.8.8
Router(dhcp-config)# exit

      2. Check that all PCs have received the correct IP addresses.

ping 192.168.1.1
